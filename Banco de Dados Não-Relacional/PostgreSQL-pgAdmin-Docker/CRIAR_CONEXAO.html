1

Fazer Login

  1. Abra o navegador
  2. Acesse http://localhost:5050
  3. Digite o email: admin@example.com
  4. Digite a senha: admin123
  5. Clique em Login
2

Criar Novo Servidor

No painel esquerdo:

  1. Clique com botão direito em Servers
  2. Selecione Create → Server
OU
Menu superior: Object → Create → Server
3

Preencher Aba "General"

Name:
PostgreSQL - testdb
Description:
Conexao local ao PostgreSQL testdb via Docker
Comment:
[opcional]
4

Preencher Aba "Connection"

Campo Valor
Host name/address postgres_db
Port 5432
Maintenance DB postgres
Username admin
Password admin123
Save password? ✓ Marque (opcional)
5

Aba "Advanced" (Opcional)

Campo Valor
SSL Mode prefer
DB restriction [deixe em branco]
Role restriction [deixe em branco]
6

Salvar Conexão

  1. Clique no botão Save
  2. Aguarde a conexão ser estabelecida
  3. Você verá a conexão aparecer no painel esquerdo

✓ Conexão Criada com Sucesso!

Se você conseguir ver o servidor "PostgreSQL - testdb" listado no painel esquerdo, a conexão está funcionando perfeitamente!

7

Testar Conexão

  1. Clique direito em testdb
  2. Selecione Query Tool
  3. Execute uma query teste:
    SELECT version();
  4. Você verá a versão do PostgreSQL como resultado

Checklist de Validação

  • Servidor "PostgreSQL - testdb" aparece no painel esquerdo
  • Databases estão visíveis (postgres, testdb, template0, template1)
  • Consegue expandir "testdb"
  • Consegue executar queries no Query Tool
  • Versão do PostgreSQL retorna sem erro

🔧 Se Houver Problemas

❌ "Could not connect to server"

  • Verifique se o PostgreSQL está rodando: docker compose ps
  • Verifique o host: deve ser postgres_db, não localhost
  • Verifique a porta: deve ser 5432

❌ "Password authentication failed"

  • Verifique a senha (case-sensitive): admin123
  • Tente desmarcar "Save password" e digitar novamente

❌ "Network unreachable"

  • Reinicie os containers: docker compose restart
  • Aguarde alguns segundos para o PostgreSQL iniciar
  • Tente conectar novamente
